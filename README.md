# RAG Policy Q&A Application (FastAPI + LangChain)

This is the **FastAPI and LangChain** implementation of the RAG Policy Q&A Application. It allows employees to ask questions about company policies and get grounded, cited answers generated by an LLM (Llama 3.3).

## Features

- **FastAPI Backend**: Modern, fast web framework with automatic validation via Pydantic.
- **LangChain RAG**: Uses LangChain Expression Language (LCEL) for ingestion, retrieval, and generation.
- **Grounded Answers**: Context is retrieved via semantic search and injected into the prompt.
- **Guardrails**: Refuses out-of-scope questions and ensures citations are provided.
- **Chat UI**: Built-in dark-themed chat interface.

## Quick Start

### 1. Setup Environment

```bash
git clone <your-repo-url>
cd ai-project-langchain

# Create virtual environment
python3 -m venv .venv
source .venv/bin/activate  # On Windows: .venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

### 2. Configuration

```bash
cp .env.example .env
```
Edit `.env` and add your `GROQ_API_KEY` (Get a free key at [console.groq.com](https://console.groq.com)).

### 3. Ingest Policy Corpus

```bash
python -m app.rag.ingest
```
This script uses LangChain loaders and splitters to parse the 10 policy documents in `corpus/`, embed them using `HuggingFaceEmbeddings` (running locally on CPU), and store them in a persistent ChromaDB instance.

### 4. Run the Server

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```
Open [http://localhost:8000](http://localhost:8000) to use the chat interface.

## API Endpoints

- `GET /` - Serve the React/vanilla HTML chat UI.
- `POST /chat` - Expects `{"question": "..."}`, returns `{"answer": "...", "citations": [...]}`.
- `GET /health` - Application health check.

## Evaluation

```bash
python evaluation/evaluate.py
```
This runs 25 questions across 10 policy categories to measure groundedness, citation accuracy, and request latency. Results are saved to `evaluation/results_langchain.json`.
